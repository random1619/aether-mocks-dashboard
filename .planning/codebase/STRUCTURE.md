# Codebase Structure

**Analysis Date:** 2026-06-29

## Directory Layout

```text
[project-root]/
├── .firebase/                  # Firebase CLI configuration cache
├── .planning/                  # GSD planning and codebase intelligence directory
│   └── codebase/               # Codebase documentation (where this file lives)
├── public/                     # Public web assets folder
│   ├── 360 Mocks/              # HTML mock tests from 360 Mocks
│   ├── English Madhyam/        # HTML mock tests from English Madhyam
│   ├── Oliiveboardd/           # HTML mock tests from Oliveboard
│   ├── Pundiits/               # HTML mock tests from Pundits
│   │   ├── css/                # Compiled stylesheet artifacts for quizzes
│   │   ├── src/                # Source SCSS sheets for Pundits quizzes
│   │   ├── extract_styles.py   # Python utility to move inline styles
│   │   ├── organize_subjects.py# Python utility to structure quizzes
│   │   ├── package.json        # Sass preprocessor build configuration
│   │   └── package-lock.json   # Sass compilation dependencies lockfile
│   ├── static GK/              # Static general knowledge HTML materials
│   ├── The solver/             # HTML mock tests from The Solver
│   ├── index.html              # Main dashboard HTML & script client
│   └── mocks-data.js           # Compiled Javascript dataset for mocks index
├── .firebaserc                 # Firebase environment configuration targets
├── firebase.json               # Firebase deployment static configuration
├── generate_mocks_data.py      # Python script to parse HTML directories & compile index
└── render.yaml                 # Render static site deployment configuration
```

## Directory Purposes

**public/**
- Purpose: Root web directory containing all user-facing content (HTML files, stylesheet directories, global javascript).
- Contains: `index.html` (dashboard entry), `mocks-data.js` (database file), and provider subdirectories with individual mock tests.

**public/Pundiits/**
- Purpose: Holds quizzes, scripts, and SCSS compilations for Pundits mock tests.
- Contains: Python scripts for local cleanup (`extract_styles.py`, `organize_subjects.py`), and CSS build configuration (`package.json`, `src/scss`).

**public/360 Mocks/, public/English Madhyam/, public/Oliiveboardd/, public/The solver/, public/static GK/**
- Purpose: Vendor folders containing static HTML quizzes that represent individual mock tests.

**.planning/**
- Purpose: GSD Core project management directory, containing milestones, ROADMAP, requirements, and codebase mappings.

## Key File Locations

**Entry Points:**
- `public/index.html`: Entry point for the Aether Mocks dashboard.
- `generate_mocks_data.py`: CLI python script to index mock documents.

**Configuration:**
- `firebase.json`: Configures deployment to Firebase hosting.
- `render.yaml`: Configures deployment to Render.
- `public/Pundiits/package.json`: Configurations for sass compiler commands.

**Core Data Files:**
- `public/mocks-data.js`: The metadata listing generated by `generate_mocks_data.py`.

## Naming Conventions

**HTML Quizzes:**
- Quizzes generated by vendor scrapes or cleanups. Often contain prefix `@the_solvers_` or suffix numbers like `_78060.html`.
- Normalized by the compiler script into human-readable space-separated labels.

**Python Scripts:**
- Snake-case naming (e.g. `generate_mocks_data.py`, `extract_styles.py`, `organize_subjects.py`).

## Where to Add New Code

**Adding a New Provider/Mock Files:**
- Place files in `public/<Provider Name>/<Category Name>/[Files].html`.
- Run `python generate_mocks_data.py` to index the new files and update the dashboard.

**Modifying Dashboard Design/Logic:**
- Edit `public/index.html` styles or JS controller.
